# ✅ Проект №2: Система управления пассажиропотоком
## 🎯 Цель
- Оптимизация перевозок за счёт анализа и прогнозирования пассажиропотока на маршрутах.

## 📌 Описание
- Система собирает данные о количестве входов/выходов на остановках, строит прогноз загруженности, помогает диспетчеру корректировать график.
- Интегрируется с мобильным приложением пассажира ("Следующий автобус будет переполнен").

## 🧱 Модели
```
public class PassengerCount {
    private Long id;
    private Bus bus;
    private Stop stop;
    private Integer entered;
    private Integer exited;
    private LocalDateTime timestamp;
}

public class Bus {
    private Long id;
    private String model;
    private Route route;
}

public class Stop {
    private Long id;
    private String name;
    private Double lat;
    private Double lon;
}

public class Route {
    private Long id;
    private List<Stop> stops;
    private List<Bus> buses;
}

public class RoutePredictionDTO {
    private Route route;
    private LocalDateTime departureTime;
    private Integer predictedLoad; // % загруженности. Среднее значение в это время на этом маршруте + процент от дождь есть или нет
}
// Среднее кол-во пассажиров в автобусе в определенное время рассчитывается как средняя сумма (Вошли - Вышли) за весь день до 
// этого времени за все такие дни
```

## 🌐 API
- POST /api/passengers – данные с датчика учёта
- GET /api/predictios?route=7A&time=15:00&stop=Кампи - прогноз
- GET /api/predictions?route=7A – прогноз на день, по каждой остановке на каждый час
- GET /api/stops/nearby?lat=...&lon=... – ближайшие остановки

## 🚀 Доп
- Тепловая карта маршрута (цвет — уровень загруженности) Остановки от времени
- Выгрузка отчета за день, с графиками
